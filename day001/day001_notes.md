# Day 001 - 컴퓨터는 코드를 어떻게 실행하는가

## 오늘의 목표
- 프로그램, 프로세스, 스레드의 차이를 설명할 수 있다
- 코드가 실행될 때 메모리와 CPU에서 무슨 일이 일어나는지 이해한다

## 핵심 개념 요약

## 프로그램 / 프로세스 / 스레드

### 프로그램 (Program)
- 디스크(SSD/HDD)에 저장된 실행 파일 또는 코드
- 실행되기 전까지는 아무 자원도 사용하지 않는다.

### 프로세스 (Process)
- 실행 중인 프로그램
- 메모리에 적재되고 CPU 스케줄링 대상이 된다.
- 운영체제가 관리하는 실행 단위

### 스레드 (Thread)
- 프로세스 내부에서 실제로 실행되는 작업 단위
- 하나의 프로세스는 최소 1개 이상의 스레드를 가진다.
- 여러 스레드는 자원을 공유하며 동시에 실행될 수 있다.

### Chrome 예시
- Chrome 프로그램: 1개
- Chrome 프로세스: 여러 개 (탭, 렌더링, 확장 프로그램 등)
- 각 프로세스 내부에 여러 스레드 존재

## CPU / RAM / Disk 역할

- Disk: 데이터를 영구 저장하는 공간 (느림)
- RAM: 실행 중인 데이터를 임시 저장하는 공간 (빠름, 휘발성)
- CPU: 연산을 수행하는 장치 (저장 기능 없음)

### 실행 흐름
- 코드는 디스크에 저장된다.
- 실행 시 RAM으로 로드된다.
- CPU는 RAM에 있는 명령어를 실행한다.

### 왜 중요한가?
- 성능 문제의 원인을 추론할 수 있다.
- 프론트엔드 렌더링, 서버 응답 속도와 직결된다.

## 프로세스 메모리 구조

- Code: 실행할 명령어가 저장된 영역
- Data: 전역/static 데이터
- Heap: 동적 할당 영역 (객체, 배열)
- Stack: 함수 호출과 매개변수, 지역 변수 저장

### 오류와의 연결
- Stack Overflow: 함수 호출 깊이 초과
- Memory Leak: Heap에서 참조 해제 실패

## 스택 프레임 (Stack Frame)

- 함수가 호출될 때 스택에 생성되는 실행 정보 묶음
- 하나의 함수 호출 = 하나의 스택 프레임

### 스택 프레임 구성
- 매개변수
- 지역 변수
- 반환 주소
- 이전 프레임 정보

### 함수 호출 흐름
- 함수 호출 시 스택 프레임 push
- 함수 종료 시 스택 프레임 pop

### Heap과의 관계
- 객체는 Heap에 생성된다.
- Stack에는 Heap 객체의 참조가 저장된다.
