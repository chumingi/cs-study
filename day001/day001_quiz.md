# Day 001 - Self Check Quiz

1. 프로그램과 프로세스의 차이를 설명하라.
2. 디스크에서 바로 실행하지 않고 RAM으로 올리는 이유는?
3. 스택과 힙의 차이를 예시와 함께 설명하라.
4. 스레드가 여러 개면 왜 빠를 수 있는가?
5. 이 개념이 프론트엔드 또는 백엔드에서 왜 중요한가?

---

## Q1-1. 프로그램과 프로세스의 차이는?

### 내 답변
- 프로그램: 디스크에만 존재하는 코드
- 프로세스: 실행 중인 프로그램

## Q1-2. Chrome 탭 하나에 프로세스가 여러 개인 이유는?

### 내 답변
- 안정성과 성능
- 탭 하나가 죽어도 브라우저 전체가 죽지 않도록 하기 위해서이다.

## Q1-3. 스레드가 없으면 어떤 문제가 생기는가?

### 내 답변
- 하나의 작업이 끝나기 전까지 다른 작업들은 대기하고 있어야 한다.

## Q1-4. 이 개념이 서버 성능과 어떤 관련이 있는가?

### 내 답변
- 서버 응답 속도가 늦어져 버벅이는 문제가 발생한다.

---

## Q2-1. 왜 디스크에서 바로 실행하지 않는가?

### 내 답변
- CPU의 처리 속도는 매우 빠르지만 디스크는 느리다.
- 디스크에서 바로 실행하면 매우 비효율적이다.

## Q2-2. RAM이 휘발성 메모리인 이유는?

### 내 답변
- 실행할 코드를 임시 저장하는 공간이기 때문에 영구 저장할 필요가 없다.

## Q2-3. CPU가 저장을 담당하지 않는 이유는?

### 내 답변
- CPU가 저장까지 담당하면 구조가 복잡해지고 속도가 느려지기 때문이다.
- 저장은 디스크나 RAM이 담당하고, CPU는 연산만 담당해야 한다.

## Q2-4. 이 구조가 서버 성능과 어떤 관련이 있는가?

### 내 답변
- 서버 요청이 증가하면 RAM에 옮겨져야 할 데이터 및 CPU 연산이 늘어난다.
- RAM이 부족해지면 스왑이 발생하여 서버가 느려진다.
- CPU 병목이 발생하면 요청이 대기하게 되고, 응답이 늦어진다.

## Q3-1. 함수 호출 시 stack에는 무엇이 저장되는가?

### 내 답변
- 스택 프레임이 저장된다.
- 스택 프레임에는 매개변수, 반환할 주소, 이전 스택 프레임 정보 등이 포함된다.

## Q3-2. 객체와 배열은 왜 Heap에 저장되는가?

### 내 답변
- 객체와 배열은 크기가 변하기 때문에 동적으로 메모리를 할당하는 Heap에 저장되어야 한다.

## Q3-3. Stack Overflow는 왜 발생하는가?

### 내 답변
- 함수가 호출되면 스택에 쌓인다.
- 호출한 함수를 종료하지 않고 계속 다른 함수를 호출하면 스택의 공간이 없어져 Stack Overflow가 발생한다.

## Q3-4. 메모리 누수가 Heap 문제인 이유는?

### 내 답변
- Heap에 객체를 생성하고 참조를 해제하지 않으면 Garbage Collector의 대상이 되지 않는다.
- Heap 사용량이 계속 증가하면 메모리 누수가 발생한다.

## Q3-5. 함수 호출과 스택 프레임의 관계를 설명하라.

### 내 답변
- 함수가 호출되면 스택 프레임이 생성되어 스택에 push된다.
- 함수가 종료되면 스택에서 pop하여 제거된다.

## Q3-6. 객체를 생성할 때 Stack과 Heap에는 각각 무엇이 저장되는가?

### 내 답변
- Heap에는 객체가 저장된다.
- Stack에는 해당 객체의 참조 주소가 저장된다.

## Q3-7. 콜스택이란 무엇인가?

### 내 답변
- Stack 영역에 스택 프레임이 쌓여 있는 상태이다.

## Q3-8. 에러 로그의 함수 호출 순서를 어떻게 해석해야 하는가?

### 내 답변
- 에러 로그에는 콜스택 정보가 포함된다.
- 가장 위에 나오는 함수가 에러 발생 지점이다.

## Q3-9. 재귀 함수가 Stack Overflow를 일으키는 이유는?

### 내 답변
- 재귀 함수는 계속 해서 함수를 호출하고, 함수 호출이 종료되지 않은 상태로 Stack 영역에 계속 쌓이게 된다.
- Stack 영역의 공간이 부족해져 Stack Overflow가 발생한다.
